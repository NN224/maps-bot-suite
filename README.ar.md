# 🤖 دليل تشغيل bot-suite (عربي)

> دليل عملي: الأوامر، مراقبة اللوج الحي، وكيف تشغّل كل نوع.
> للتنصيب من الصفر شوف [README.md](README.md).

---

## 1) مشروع واحد، وضعان للتشغيل

كل شي بمشروع واحد (`bot-suite`). نفس الأمر `./bot run`، بس بتختار الوضع:

| | **ranking + تفاعل** (الافتراضي) | **رشق autocomplete** (`--burst`) |
|---|---|---|
| الأمر | `./bot run 50` | `./bot run 50 --burst` |
| السيناريو | `traffic_engage` | `pr_burst` |
| الفكرة | بحث → سكرول → نقر ملفك → تفاعل (reviews/website/directions) | يكتب بادئة قصيرة + ينقر ملفك (يغذّي الاقتراحات) |
| الكلمات | بادئات بزنسك (من الإعدادات) | مزيج موزون من بادئات بزنسك (من الإعدادات) |

> 🎯 الهدف (البزنس) + الكلمات تعيش بقاعدة البيانات و`.env` — **مش بالكود**. تتحكّم فيها بـ `./bot biz`.
> ⚠️ بوت واحد بنفس الوقت (قفل PID). لو بدك تشغّل الاثنين بالتوازي لاحقاً = ميزة منضيفها.

## 2) الأوامر

```bash
cd maps-bot-suite     # المجلد اللي طلع من git clone
# الـ ./bot بيفعّل الـ venv لحاله (ما تحتاج source)

./bot run 50                      # 50 جلسة ranking بالخلفية (headless)
./bot run 50 --burst              # 50 جلسة رشق autocomplete
./bot run 5 --visible             # 5 جلسات مع إظهار المتصفّح
./bot run 1000 --continuous       # متواصل لحد ما توقفه (بيتجاهل الرقم)
./bot run 1000 --burst --continuous --min-delay 4 --max-delay 12   # رشقة مكثّفة
./bot run 50 --free-proxies       # على بروكسيات مجانية (تجربة $0)

./bot status                      # إحصائيات اليوم
./bot dash                        # 📊 لوحة TUI حية (الأفضل للمتابعة)
./bot log                         # 📜 اللوج الحي (tail -f)
./bot stop                        # إيقاف
./bot biz                         # إدارة البزنس (list/add/switch/edit/delete)
```

## 3) مراقبة اللوج الحي (live)

```bash
./bot log                      # متابعة حية (tail -f) للوج
# أو مباشرة:
tail -f /tmp/bot_suite.log
```

**اللوحة الحية (الأفضل):** `./bot dash` — بتعرض:
- الحالة (RUNNING/idle) + البزنس الفعّال + الوقت
- إحصائيات اليوم: Total · Success% · Partial · Failed
- **📍 Avg rank / 🏆 Best rank** ← أهم KPI (ملوّن: أخضر = أعلى)
- التفاعل: Directions · Calls · Website · Reviews
- آخر الجلسات: الوقت · الحالة · **المرتبة (#)** · Dir/Cal/Web/Rev · المدّة
- **اللوج الكبير الحي** بالألوان (كل خطوة بالإنجليزي: STEP 1→7) + شريط أوامر
- تطلع بـ `Ctrl+C`

## 4) أنواع التشغيل

| النوع | الأمر | متى |
|------|------|-----|
| **ranking عادي** | `./bot run 50` | الحملة الأساسية (تفاعل كامل) |
| **رشق autocomplete** | `./bot run 1000 --burst --continuous --min-delay 4 --max-delay 12` | رفع الملف بالاقتراحات |
| **رشق مجاني** | `./bot run 1000 --burst --free-proxies --continuous` | تجربة $0 (جغرافيا عشوائية، أضعف) |
| **مرئي للفحص** | `./bot run 1 --visible` | تشوف بعينك شو بيصير |

**قبل الرشق المجاني** — جيب قائمة البروكسيات أول:
```bash
./venv/bin/python -c "from shared.free_proxy import fetch_and_store; print(fetch_and_store(candidate_limit=800,test=True,max_workers=50))"
```

## 5) البروكسي (`.env`)

- عبّي بروكسيك بـ `PROXY_URL` في `.env`. بروكسي الموبايل بيحمّل Maps أسرع وأثبت من الـ residential أو المجاني.
- كل جلسة IP جديد تلقائياً (ضع `{session}` بالرابط لو مزوّدك بيدعمه).
- لو خلص الرصيد: البوت بيوقف لحاله (proxy-down بعد ٤ فشل متتالي) → عبّي وأعد التشغيل.
- `BLOCK_MEDIA=1` (افتراضي) = حجب الصور لتوفير الباندويث (~10-16 MB/جلسة).

## 6) إضافة بزنس وقياس النتيجة

```bash
./bot biz add            # أضف ملفك (الاسم، place_id، النوع، الكلمات)
./bot biz switch <slug>  # فعّلو
./bot run 5 --visible    # جرّب وشوف
```

**قياس التجربة:**
1. **baseline:** من سطح المكتب (incognito، بمدينتك) اكتب بادئة بزنسك بـ Maps — لاحظ مرتبة ملفك. صوّرها.
2. شغّل الحملة 24-48 ساعة.
3. **أعد نفس البحث** → إذا ملفك طلع لفوق = بيشتغل ✅.

---

## 7) ملاحظات مهمة

1. **ابدأ صغير:** `./bot run 1 --visible` وراقب قبل الدفعات الكبيرة.
2. **خلّي الجهاز صاحي** أثناء التشغيل المتواصل (عطّل sleep) وإلا بتوقف.
3. النتيجة بتاخد وقت — الترتيب بيتحرّك بالاستمرارية مش بدفعة وحدة.
4. **الأدوم والأقوى = المراجعات + نشاط ملفك** — الرشق إشارة مساعِدة.
